# 直播绑定券码/促销ID

从直播业务提报表读取 SKU 和绑定值列，生成京东直播后台官方批量上传模板，并输出异常报告。

第一版只负责生成文件，不自动登录或上传京东直播后台。

## 目录

- `promotion_binding/`：绑定券码/促销 ID 的核心逻辑。
- `assets/`：京东后台官方商品上传模板。
- `input/`：本地测试或临时上传文件。
- `output/`：生成的上传模板和异常报告。
- `tests/`：单元测试。

## 核心规则

- 上传后先确认列映射；系统会自动推荐，用户也可以手动改选。
- SKU 列推荐关键词：`sku`、`skuID`、`上播 SKU ID`。
- 绑定值列推荐关键词：`券码`、`价码`、`促销编码`、`专享券`、`专享价`、`达人id`。
- 识别 `vender_BA#...` 作为专享券 KEY。
- 识别纯数字长 ID 作为专享价促销 ID。
- 同一个 SKU 同一行不能同时写专享券 KEY 和专享价促销 ID。
- 无效文本、重复 SKU、多值冲突进入异常报告。

## 异常报告口径

- `汇总`：展示可上传、KEY、促销 ID、空值跳过、异常、重复的数量。
- `可上传明细`：展示原始行号、SKU、商品名称、绑定类型、绑定值、写入模板行号。
- `需处理异常`：展示需要业务人工确认的问题，例如无效文本、多 KEY、多促销 ID、同 SKU 多绑定值。
- `跳过和重复`：展示空值跳过和重复跳过；重复行会标明保留的原始行号。

## 输出文件名

- `京东绑券上传模板_YYYYMMDD-HHmmss_.xlsx`
- `异常报告_YYYYMMDD-HHmmss_.xlsx`
