## 上播·SKU ID

- 就是正常的数字形式的 SKU ID

    - 比如：19014258025

## 促销券和促销编码

- 绑定的内容分三种

    - 第一种：促销券（口头叫这个，系统里叫做**专享价促销 ID**）

        - 数字形式：395442423188

    - 第二种：促销编码（口头叫这个，系统里叫做**专享价的 KEY**）

        - 一般英文开头：vender\_BA\#a9d94c41368e441094132b17a3b40fd6

    - 第三种：另一种促销 KEY（系统识别为**专享券 KEY**）

        - `BA_` 开头，后跟 6 位及以上字母数字，例如：BA\_9t7zua1、BA\_9bz63kn

- 【券码/价码达人id：22766602】列有效信息就促销券、促销编码和 BA_ 开头促销 KEY 三种，可能包含其他中文等信息，是不用考虑的

## 短卖点

业务提报表格中可能包含一列短卖点信息，用于写入京东直播后台商品上传模板的【利益点（非必填，不超过22个字符）】列（官方模板 B 列）。

- 系统会按以下关键词自动识别短卖点列：`短卖点`、`利益点`、`卖点`。
- 识别到后，该列内容会作为对应 SKU 的短卖点写入输出模板；若未识别到，会在页面状态提示“未识别到短卖点列”。
- 短卖点超过 22 个字符时，会作为“短卖点警告”写入异常报告，但仍保留原值进入输出模板。
- 只有短卖点、没有券码/促销编码的行，也会进入输出模板。
- 同一个 SKU 出现多次且短卖点不一致时，保留第一条短卖点。

促销编码绑定是有 3 种

1. 第一种：每次输入一条 sku

2. 第二种：可批量输入 SKU，每次最多50个，用英文“，”隔开

3. 第三种：基于平台提供的 Excel模板，批量导入
