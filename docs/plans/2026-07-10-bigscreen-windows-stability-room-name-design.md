# 蓝屏截图 Windows 稳定性与直播间名称监控设计

## 背景

同一直播大屏截图流程在 macOS 正常，在 Windows 打包版中大量出现“未找到页面元素”“未找到下拉框”和“未找到商品分析表头”。失败步骤通常在 3 到 4 秒内结束，而现有实现每次切换页面固定等待 1.5 秒后便立即用元素数量判断是否存在。

实页检查还确认：

- 左侧相同页面在多个截图步骤之间会被重复点击，导致内容区域反复重建。
- Windows 打包环境渲染较慢，目标元素可能尚未挂载就被判定为不存在。
- 现有登录检查只验证京东购物车页，没有验证直播大屏页面是否真正可用。
- 直播间账号名称位于大屏右上角用户名节点，例如“京东青春采销”。

## 目标

- Windows 页面加载较慢时，截图流程等待目标元素出现，不提前报错。
- 已选中的左侧页面不重复点击，减少页面重建和网络请求。
- 登录状态需要通过直播大屏核心元素验证，避免购物车登录态误判。
- 蓝屏截图完成监控携带右上角账号名称 `extra.room_name`。
- 当前会话下载监控在已知名称时也携带 `extra.room_name`。

## 非目标

- 不调整 15 个截图项和截图区域。
- 不修改远端监控表结构或接口。
- 不把本场直播标题作为 `room_name`。
- 不强制 `task_start` 携带名称，因为此时浏览器尚未打开大屏页面。

## 设计

### 页面等待

页面元素定位从“立即统计数量”改为显式等待可见状态。等待超时后继续使用现有中文业务错误，但补充页面就绪错误信息，便于区分登录态或权限问题与单个元素缺失。

### 幂等导航

左侧导航使用侧栏名称节点定位。若对应侧栏容器已经带有选中状态，则直接复用当前页面；只有目标页面未选中时才点击。流量页的多个指标因此只在第一次进入时加载一次。

### 大屏登录校验

保留现有京东登录检查作为第一层校验，随后打开目标直播大屏并等待侧栏核心节点。大屏核心节点未出现时，将登录状态视为无效，进入现有可视化登录流程。

### 直播间名称

浏览器从右上角 `[class*="header-index-currentUserName"]` 节点读取账号名称。`CaptureOnceResult` 返回 `room_name`，Web 任务状态保存该值，并在 `task_finish.extra.room_name` 与当前会话的 `download.extra.room_name` 中上报。

## 验证

- 单元测试模拟元素延迟出现，确认代码调用显式等待。
- 单元测试模拟侧栏已选中，确认不会重复点击。
- 单元测试模拟购物车已登录但大屏未就绪，确认登录校验失败。
- 服务测试确认截图结果返回右上角账号名称。
- Web 路由测试确认完成和下载监控包含 `room_name`。
- 运行蓝屏截图、Web 入口和根目录回归测试。
