# 蓝屏截图 Windows DOM 点击与状态校验设计

## 问题确认

Windows 有头模式使用 80% CSS 缩放时，Playwright 的坐标点击发生偏移：第 04 步本应点击“全部商品”下拉框，却切到了左侧“商品”页；随后切换“流量”“概览”等操作也没有生效，页面长期停留在商品页。

现有日志只验证截图文件是否写入，没有验证标签、侧栏或下拉框是否真的切换成功。因此 01 到 03 虽显示成功，截图内容仍可能不正确。后续每项又等待元素 15 秒，造成“错误页面上长时间等待”的额外耗时。

## 目标

- 保留 80% 页面缩放和现有截图视觉范围。
- 所有业务点击不再依赖屏幕坐标。
- 点击侧栏、概览标签、流量指标和下拉框后验证真实状态。
- 状态未变化时重试一次，仍失败则快速报出明确错误。
- 商品表头排序继续验证降序结果。

## 实现方案

### DOM 点击

统一通过元素自身的 `click()` DOM 事件触发交互，替换 `locator.click(force=True)`。DOM 点击不需要 Playwright 计算缩放后的屏幕坐标，因此不受 Windows DPI 与 CSS zoom 组合影响。

覆盖范围：

- 左侧概览、流量、商品导航。
- 概览页在线、流量、成交标签。
- 流量页指标标签。
- Ant Design 下拉框触发器和选项。
- 商品分析表头排序。

### 状态校验

- 侧栏：校验目标侧栏进入选中状态。
- 概览标签：校验对应单选按钮或父标签进入 checked 状态。
- 流量指标：校验目标指标进入 selected 状态。
- 下拉框：校验当前选中项文本变为目标值。
- 商品排序：沿用排序图标或可见列值降序校验。

每次点击后进行短间隔轮询。第一次未生效时再次 DOM 点击；第二次仍未生效则立即失败，不再让后续步骤在错误页面等待 15 秒。

### 兼容性

继续使用 Python 3.9 和 Playwright 1.40 可用 API。不修改 BrowserManager 的全局缩放逻辑，避免影响 SKU 测价和其他工具。

## 测试

- 证明业务点击调用 DOM `el.click()` 而不是坐标点击。
- 模拟第一次点击未生效、第二次生效。
- 模拟两次点击均未生效并验证快速错误。
- 验证侧栏、标签、指标和下拉框状态检查。
- 回归 15 个截图步骤、Web 入口和 Windows 打包静态检查。
