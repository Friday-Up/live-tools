# 用户完整操作流程及检查点

## 一、下载安装阶段

### 1.1 下载 ZIP 文件
- [ ] 用户从 GitHub Release 下载 `SKU-Price-Audit-Web-Windows.zip`
- [ ] 检查：下载是否完整，文件大小是否正常

### 1.2 解压 ZIP 文件
- [ ] 用户解压到任意目录
- [ ] 检查：解压后的文件结构是否正确
```
SKU-Price-Audit-Web/
├── SKU-Price-Audit-Web.exe    ← 主程序
├── 启动测价工具.bat            ← 启动脚本
├── 关闭服务.bat               ← 关闭脚本（启动后自动创建）
├── templates/
│   └── index.html             ← 前端页面
├── config.py                  ← 配置文件
├── utils/                     ← 工具模块
│   ├── __init__.py
│   ├── browser_manager.py     ← 浏览器管理
│   ├── jd_crawler.py          ← 京东爬虫
│   ├── excel_handler.py       ← Excel处理
│   └── cleanup.py             ← 清理工具
└── input/                     ← 输入目录（空）
```

### 1.3 准备输入文件
- [ ] 用户将 Excel 文件放入 `input/` 目录
- [ ] 检查：Excel 格式是否正确（.xlsx，包含「商品SKU」列）

---

## 二、启动阶段

### 2.1 双击启动脚本
- [ ] 用户双击 `启动测价工具.bat`
- [ ] 检查：是否弹出终端窗口？（应该隐藏）
- [ ] 检查：是否自动打开浏览器？（应该自动打开）
- [ ] 检查：打开的页面地址是否正确？（应该是 http://127.0.0.1:8080）

**可能的问题：**
1. 终端窗口没有隐藏
2. 没有自动打开浏览器
3. 打开两个页面（127.0.0.1 和 about:blank）
4. 服务启动失败，页面显示"无法访问"

---

## 三、首次使用（未登录状态）

### 3.1 上传文件
- [ ] 用户点击上传区域或拖拽文件
- [ ] 检查：文件是否成功上传
- [ ] 检查：是否显示文件名和状态

### 3.2 设置门槛价
- [ ] 用户输入价格门槛（默认6.0）
- [ ] 检查：输入框是否正常工作

### 3.3 点击"开始测价"
- [ ] 用户点击按钮
- [ ] 检查：按钮是否禁用（防止重复点击）
- [ ] 检查：是否显示进度卡片

### 3.4 启动浏览器
- [ ] 检查：是否自动打开 Chrome/Chromium 浏览器
- [ ] 检查：浏览器是否显示京东页面

### 3.5 登录检测
- [ ] 检查：后端是否检测登录状态
- [ ] 检查：如果未登录，是否显示"需要登录"弹窗
- [ ] 检查：弹窗是否有"我已登录，继续"按钮

**可能的问题：**
1. 检测不到未登录状态
2. 弹窗不显示
3. 终端显示"需要人工登录"但页面无反应

### 3.6 用户登录
- [ ] 用户在浏览器中完成京东登录（扫码或密码）
- [ ] 检查：登录成功后页面是否显示用户信息

### 3.7 点击"我已登录，继续"
- [ ] 用户点击按钮
- [ ] 检查：弹窗是否关闭
- [ ] 检查：测价是否继续执行

**可能的问题：**
1. 点击后无反应
2. 提示"登录失败"
3. 测价没有继续

---

## 四、测价执行阶段

### 4.1 进度显示
- [ ] 检查：进度条是否正常更新
- [ ] 检查：当前/总数是否正确显示
- [ ] 检查：低于门槛数量是否正确统计

### 4.2 日志显示
- [ ] 检查：是否显示实时日志
- [ ] 检查：日志内容是否正确（SKU、价格、状态）

### 4.3 浏览器行为
- [ ] 检查：浏览器是否自动跳转商品页面
- [ ] 检查：是否自动遍历规格
- [ ] 检查：是否正确截图

---

## 五、停止测价

### 5.1 点击"停止测价"
- [ ] 用户点击按钮
- [ ] 检查：是否弹出确认对话框
- [ ] 检查：确认后测价是否停止
- [ ] 检查：浏览器是否关闭
- [ ] 检查：进度是否停止更新

**可能的问题：**
1. 点击后无反应
2. 浏览器没有关闭
3. 后端仍在运行
4. 再次点击"开始测价"无法启动

---

## 六、测价完成

### 6.1 完成显示
- [ ] 检查：是否显示"测价完成"
- [ ] 检查：是否保留进度统计（总数、低于门槛数等）
- [ ] 检查：是否显示"下载结果"按钮

### 6.2 下载结果
- [ ] 用户点击"下载结果"
- [ ] 检查：是否下载 Excel 文件
- [ ] 检查：Excel 内容是否正确

### 6.3 再次测价
- [ ] 用户点击"再次测价"
- [ ] 检查：是否重置进度
- [ ] 检查：是否可以重新上传文件

---

## 七、关闭服务

### 7.1 关闭方式
- [ ] 方式1：点击页面上的"关闭服务"按钮
- [ ] 方式2：双击"关闭服务.bat"
- [ ] 方式3：任务管理器结束进程

### 7.2 检查
- [ ] 检查：服务是否完全关闭
- [ ] 检查：浏览器是否关闭
- [ ] 检查：端口 8080 是否释放

---

## 八、二次使用（已登录状态）

### 8.1 重新启动
- [ ] 用户再次双击启动脚本
- [ ] 检查：是否自动加载上次登录态
- [ ] 检查：是否跳过登录步骤

### 8.2 登录态失效
- [ ] 如果登录态过期
- [ ] 检查：是否正确检测到失效
- [ ] 检查：是否显示登录弹窗

---

## 九、异常情况

### 9.1 无输入文件
- [ ] 不上传文件直接点击开始
- [ ] 检查：是否有错误提示

### 9.2 格式错误
- [ ] 上传非 xlsx 文件
- [ ] 上传不含「商品SKU」列的 Excel
- [ ] 检查：是否有错误提示

### 9.3 网络异常
- [ ] 断网情况下启动测价
- [ ] 检查：是否有错误提示

### 9.4 浏览器异常
- [ ] 手动关闭浏览器窗口
- [ ] 检查：程序是否能正常处理
